# Поиск и фильтрация отелей

## Обзор функциональности

Этот документ описывает реализованную систему поиска и фильтрации отелей в приложении HotelApp.

---

## Компоненты системы

### 1. Экран поиска (SearchScreen)

**Путь:** `Home Screen → [🔍] → Search Screen`

**Элементы управления:**

| Элемент | Описание |
|---------|----------|
| **Выбор города** | Выпадающий список с городами из mock-данных |
| **Выбор дат** | Кастомный календарь для выбора диапазона (заезд/выезд) |
| **Количество гостей** | Счётчик с кнопками +/- |
| **Кнопка "Найти отели"** | Переход на экран результатов |

#### Выбор города
- Города подгружаются из `AppRepository.getAllCities()`
- Основаны на реальных городах, где есть отели
- Опция "Все города" для сброса фильтра

#### Кастомный DatePicker
- Двухэтапный выбор: сначала дата заезда, потом выезда
- Визуальное выделение выбранного диапазона
- Навигация по месяцам (стрелки влево/вправо)
- Кнопки "Сброс" и "OK"
- Валидация: дата заезда не может быть в прошлом

#### Выбор гостей
- Кнопки "+" и "-" для изменения количества
- Минимум: 1 гость
- Отображение: "N гост."

---

### 2. Экран результатов (SearchResultsScreen)

**Путь:** `Search Screen → [Найти отели] → Search Results Screen`

**Элементы:**

| Элемент | Описание |
|---------|----------|
| **TopAppBar** | Название + кнопки фильтров и сортировки |
| **Счётчик результатов** | "N отелей найдено" |
| **Активные фильтры** | Чипсы с возможностью удаления |
| **Список карточек** | Карточки отелей с основной информацией |

#### Карточка отеля
- Фотография отеля
- Название
- Рейтинг (звезда + числовое значение + количество отзывов)
- Город и тип жилья
- Цена ("от N ₽ за ночь")
- Кнопка "Подробнее" → переход на экран деталей

---

### 3. Панель фильтров (Bottom Sheet)

**Открытие:** Кнопка `[⚙️ Фильтры]` в TopAppBar

**Параметры фильтрации:**

#### 3.1 Диапазон цен
- **Двухпозиционный ползунок** для выбора мин/макс цены
- **Текстовые поля** для ручного ввода
- Автоматическая синхронизация ползунка и полей
- Диапазон определяется из данных отелей (min..max)

#### 3.2 Тип жилья
- Горизонтальный список чипсов
- Мультивыбор
- Типы из `HotelType`:
  - Отель
  - Курорт
  - Апартаменты
  - Хостел
  - Гостевой дом
  - Вилла

#### 3.3 Удобства
- Вертикальный список с чекбоксами
- Данные подгружаются из `AppRepository.getAllAmenities()`
- Извлекаются из всех отелей (уникальные значения)
- Мультивыбор

**Текущие удобства в mock-данных:**
```
• Wi-Fi
• Бассейн
• Спа
• Ресторан
• Фитнес-центр
• Парковка
• Консьерж
• Пляж
• Детская площадка
• Анимация
• Камин
• Лыжная комната
• Баня
• Трансфер
• Бизнес-центр
• Бар
• Эко-тропы
• Велопрокат
• Рыбалка
• Сауна
• Кухня
• Терраса
• Вид на горы/море/озеро
• Балкон
• Мини-бар
• Сейф
• Кондиционер
• Джакузи
```

#### 3.4 Минимальный рейтинг
- Предустановленные значения: 3+, 3.5+, 4+, 4.5+
- Одиночный выбор (radio button поведение)
- Фильтр по рейтингу отеля (float 0.0-5.0)

#### 3.5 Кнопки действий
- **"Сбросить"** — сброс всех фильтров к значениям по умолчанию
- **"Применить"** — применение фильтров и обновление результатов

---

### 4. Сортировка результатов

**Открытие:** Кнопка `[⇅ Сортировка]` в TopAppBar

**Опции сортировки:**

| Опция | Описание |
|-------|----------|
| **Цена: по возрастанию** | Сначала дешёвые отели |
| **Цена: по убыванию** | Сначала дорогие отели |
| **Рейтинг: по убыванию** | Сначала отели с высоким рейтингом |
| **Отзывы: по убыванию** | Сначала отели с большим количеством отзывов |

**Поведение:**
- Выбор опции сразу применяет сортировку
- Текущая опция отмечается галочкой ✓
- Сортировка применяется после фильтрации

## Архитектура

### SearchViewModel

**Расположение:** `ui/viewmodel/SearchViewModel.kt`

**Основные методы:**

```kotlin
// Выбор параметров поиска
fun selectCity(city: String?)
fun setCheckInDate(date: String)
fun setCheckOutDate(date: String)
fun setGuests(count: Int)

// Фильтры
fun setPriceRange(min: Int, max: Int)
fun toggleHotelType(type: HotelType)
fun toggleAmenity(amenity: String)
fun setMinRating(rating: Float)
fun setMinStars(stars: Int)

// Сортировка
fun setSortOption(sortOption: SortOption)
fun toggleSortMenu()

// Управление UI
fun toggleFilters()
fun resetFilters()
fun performSearch()
```

**SearchUiState содержит:**
- `availableCities` — список городов
- `availableAmenities` — список удобств
- `selectedCity`, `checkInDate`, `checkOutDate`, `guests` — параметры поиска
- `priceRangeStart`, `priceRangeEnd` — диапазон цен
- `selectedHotelTypes`, `selectedAmenities` — выбранные фильтры
- `minRating`, `minStars` — минимальные значения
- `currentSortOption` — текущая сортировка
- `showFilters`, `showSortMenu` — состояние UI
- `searchResults` — результаты поиска
- `hasSearched` — был ли выполнен поиск
- `isLoading` — индикатор загрузки
- `error` — сообщение об ошибке

## Компоненты UI

### DateRangePicker

**Расположение:** `ui/components/DateRangePicker.kt`

**Особенности:**
- Кастомный календарь для выбора диапазона дат
- Визуальное выделение:
  - Дата заезда — скруглённый левый край
  - Дата выезда — скруглённый правый край
  - Диапазон между датами — выделен цветом
- Навигация по месяцам
- Отображение выбранных дат внизу
- Кнопки "Сброс" и "OK"

### DualThumbRangeSlider

**Расположение:** `ui/components/RangeSlider.kt`

**Особенности:**
- Два ползунка для выбора диапазона
- Drag-жесты для перемещения ползунков
- Активная область между ползунками выделяется цветом
- Отображение мин/макс значений

## Навигация

```kotlin
// Screen.kt
object Search : Screen("search")
object SearchResults : Screen("search_results")

// AppNavigation.kt
composable(Screen.Search.route) {
    SearchScreen(
        onNavigateBack = { navController.popBackStack() },
        onNavigateToResults = { navController.navigate(Screen.SearchResults.route) }
    )
}

composable(Screen.SearchResults.route) {
    SearchResultsScreen(
        onNavigateBack = { navController.popBackStack() },
        onNavigateToHotelDetails = { hotelId ->
            navController.navigate(Screen.HotelDetails.createRoute(hotelId))
        }
    )
}
```

## Примеры использования

### Сценарий 1: Поиск отеля в конкретном городе

1. Открыть экран поиска
2. Выбрать город из выпадающего списка (например, "Москва")
3. Выбрать даты заезда и выезда
4. Нажать "Найти отели"
5. Просмотреть результаты

### Сценарий 2: Поиск с фильтрами

1. Открыть экран поиска
2. Выбрать параметры
3. Нажать "Найти отели"
4. На экране результатов нажать `[⚙️ Фильтры]`
5. Выбрать:
   - Диапазон цен: 10000-20000 ₽
   - Тип жилья: Отель
   - Удобства: Wi-Fi, Бассейн
   - Рейтинг: 4+
6. Нажать "Применить"
7. Просмотреть отфильтрованные результаты

### Сценарий 3: Сортировка по цене

1. На экране результатов нажать `[⇅ Сортировка]`
2. Выбрать "Цена: по возрастанию"
3. Результаты переупорядочиваются (сначала дешёвые)

## Валидация и ошибки

| Ситуация | Сообщение | Поведение |
|----------|-----------|-----------|
| Дата заезда в прошлом | "Дата заезда не может быть в прошлом" | Не позволяет выбрать дату |
| Дата выезда раньше заезда | "Дата выезда должна быть позже даты заезда" | Не позволяет выбрать дату |
| Гостей < 1 | "Количество гостей должно быть не менее 1" | Не позволяет уменьшить |
| Ничего не найдено | "Ничего не найдено. Попробуйте изменить параметры поиска" | Пустой список с сообщением |
